Privacy Policy — Browser API Client

Last updated: 2026-03-06

Data Collection

Browser API Client does not collect, transmit, or share any personal data.

Permissions Explained

  • storage: Used to save your request history, environments, collections, and preferences locally on your device.
  • host_permissions (<all_urls>): Required to send HTTP requests to any API endpoint. The extension acts as an HTTP client — it must be able to reach any URL you choose to test. Requests are only sent when you explicitly click "Send".

How API Requests Work

When you send an API request, it is executed directly from your browser's background service worker. The request goes from your browser to the target URL you specified — no proxy, no intermediary server. Request and response data stays in your browser.

What the extension does NOT do

  • Does not collect analytics or usage data
  • Does not use cookies or tracking pixels
  • Does not proxy requests through any server
  • Does not access your browsing history
  • Does not read or modify page content

Pro License

If you purchase a Pro license, the transaction is processed by Stripe through ExtensionPay. We do not store your payment information. Your license status is stored locally in your browser.

Data Storage

Request history, environments, and collections are stored locally on your device using the browser's storage API. You can export and delete this data at any time from the extension's settings page. No data is stored on external servers.

Changes to This Policy

If this privacy policy changes, the updated policy will be posted here and the "last updated" date will be revised.

Contact

If you have questions about this privacy policy, please file an issue on the extension's support page.
